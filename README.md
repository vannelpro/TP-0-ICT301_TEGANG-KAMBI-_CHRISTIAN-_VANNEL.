*Compréhension des principes SOLID

- SRP : Une classe ne doit avoir qu’une seule responsabilité.
- OCP : Une classe doit être ouverte à l’extension mais fermée à la modification.
- LSP : Une sous-classe doit pouvoir remplacer sa classe mère sans erreur.
- ISP : Il vaut mieux plusieurs interfaces spécifiques qu’une seule interface générale.
- DIP : Les modules de haut niveau ne doivent pas dépendre des modules de bas niveau.
